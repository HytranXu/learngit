github越来越为大家所熟悉。如果发现了某个项目的BUG，也发扬一下开源精神，将修正后的代码提交给原作者。那么，本文讲得就是如何将修改后的代码向github提交。

总的流程如下：

1.先 fork 相应的项目，相当于拷贝一份到自己的github账户下

2.然后再clone 到本地分支，对bug做相应的修改

3.发起 pull request 给原作者，让他看到，并决定是否接受你的修改

4.原作者同意接受这个bug，就会 merge 到他自己的项目中

5.提交流程结束了

以 Github 上的的 bootstrap为例，实操如下：
